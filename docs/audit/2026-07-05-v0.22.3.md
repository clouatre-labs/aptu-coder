## Purpose

Point-in-time audit of `aptu-coder` v0.22.3 against four goals:

1. **rmcp 1.8.0 alignment**: verify all tool macros, transport wiring, and response patterns match the current API.
2. **Correctness**: confirm no regression from recent fixes (PR #1255 file locking, #1257 log consumer, #1268 output_truncated sync).
3. **Security**: ensure no shell execution surface or profile contract violations.
4. **CI/supply-chain hygiene**: runner architecture, action pinning, dependency advisory coverage.

Each confirmed finding includes source evidence, a fix, and a regression gate. The audit is the source material for the GitHub issues listed in the Summary.

- Five delegates in sequence: four parallel read-only research agents (rmcp alignment, core engine, exec\_command, CI/infra), then one adversarial validation delegate that cross-referenced rmcp 1.8.0 via Context7 and verified every finding against source.
- Three initial findings dismissed as false positives after source verification (see Non-Findings).
- All line references verified against HEAD (`61674fd`).

## Findings

### F02 -- AMD64 `mcp-publisher` binary on ARM64 runner

**Severity:** High
**File:** `.github/workflows/publish-registry-manual.yml:101--103`
**Verdict:** CONFIRMED

The job runs on `ubuntu-24.04-arm` (ARM64) but downloads `mcp-publisher_linux_amd64.tar.gz`. The SHA256 check passes (correct hash for the AMD64 artifact), then the binary fails at execution with "Exec format error". `publish-registry.yml` and `release.yml` correctly use the ARM64 artifact.

**Resolution:** Fixed in [#1280](https://github.com/clouatre-labs/aptu-coder/pull/1280). Download switched to `mcp-publisher_linux_arm64.tar.gz` with updated SHA256.

---

### F03 -- `rust-cache` SHA mismatch across workflows

**Severity:** Medium
**File:** `.github/workflows/build-and-attest.yml:64`
**Verdict:** CONFIRMED

`build-and-attest.yml` pins `Swatinem/rust-cache` to `e18b497796c12c097a38f9edb9d0641fb99eee32` (comment: `# v2`, a floating major-version SHA). All three references in `ci.yml` use `c19371144df3bb44fab255c43d04cbc2ab54d1c4` (comment: `# v2.9.1`). Mismatched SHAs produce different cache key formats, reducing cache hit rates between the attestation build and CI builds.

**Resolution:** Fixed in [#1280](https://github.com/clouatre-labs/aptu-coder/pull/1280). `build-and-attest.yml` pinned to `c19371144df3bb44fab255c43d04cbc2ab54d1c4 # v2.9.1`.

### F04 -- CodeQL JVM incompatible with ARM64

**Severity:** Medium
**File:** `.github/workflows/codeql.yml:22`
**Verdict:** CONFIRMED (resolution differs from original fix)

`runs-on: ubuntu-24.04` (AMD64). The original fix proposed changing to `ubuntu-24.04-arm`, but implementation revealed that CodeQL CLI v2.25.6 bundles an AMD64 JVM that cannot execute on `linux/arm64`. The runner was reverted to `ubuntu-24.04`; this is the correct long-term state until CodeQL ships a native ARM64 JVM.

**Resolution:** Reverted to `ubuntu-24.04` in [#1280](https://github.com/clouatre-labs/aptu-coder/pull/1280). CodeQL remains on AMD64 by necessity.

### F11 -- `expect_used` not covered by workspace deny lint

**Severity:** Medium
**File:** `crates/aptu-coder-core/src/graph.rs:82,174`
**Verdict:** CONFIRMED

Production code uses `.expect("len==1")` on match arms where the invariant is locally guaranteed but not statically provable. The workspace lint `clippy::unwrap_used = "deny"` covers `.unwrap()` but not `.expect()`. `clippy::expect_used` is absent from `[workspace.lints.clippy]`. If the invariant is ever violated (e.g., by a tree-sitter grammar change), the server panics at runtime rather than returning an error.

**Resolution:** Fixed in [#1282](https://github.com/clouatre-labs/aptu-coder/pull/1282). Added `clippy::expect_used = "deny"` to `[workspace.lints.clippy]`; call sites annotated with `#[allow(clippy::expect_used)]` and `// SAFETY:` comments where the invariant is locally guaranteed.

### F07 -- `cargo deny check` omits `bans` evaluation

**Severity:** Low
**File:** `deny.toml:25--27` / `.github/workflows/ci.yml`
**Verdict:** CONFIRMED

CI runs `cargo deny check advisories licenses` without `bans`. The bans section (`multiple-versions = "warn"`) is configured but never evaluated in CI. Multiple-version warnings from duplicate transitive dependencies are silently ignored.

**Resolution:** Fixed in [#1281](https://github.com/clouatre-labs/aptu-coder/pull/1281). CI command expanded to `cargo deny check advisories licenses bans`.

### F10 -- Interleaved overflow slot collision under concurrency

**Severity:** Low
**File:** `crates/aptu-coder/src/tools/exec_command.rs:~820`
**Verdict:** CONFIRMED

Interleaved output overflow uses `slot = seq % 8`, mapping all concurrent requests to one of 8 fixed slot directories. The overflow file within each slot is named `interleaved` (not `interleaved-{seq}`). Two concurrent requests mapping to the same slot that both trigger overflow write to the same path, silently overwriting each other's data. Under load (more than 8 concurrent `exec_command` calls with large interleaved output), this produces corrupted overflow data with no error surfaced.

**Resolution:** Fixed in [#1283](https://github.com/clouatre-labs/aptu-coder/pull/1283). Slot identifier changed to the full `seq` value; concurrent overflow regression test added.

---

### F12 -- Mutex poison recovery is silent in telemetry

**Severity:** Low
**File:** `crates/aptu-coder-core/src/cache.rs` (`lock_or_recover`)
**Verdict:** CONFIRMED

`lock_or_recover` recovers from a poisoned mutex via `poisoned.into_inner()`. Recovery is correct for a pure read-through cache (a cache miss, not data corruption). However, mutex poisoning indicates a prior panic in a critical section; this event is invisible in telemetry, making silent failure states undetectable in production.

**Resolution:** Fixed in [#1282](https://github.com/clouatre-labs/aptu-coder/pull/1282). Added `tracing::warn!` on the poison recovery path.

---

### F13 -- `lock_shard_shared` no-lock fallback undocumented at call sites

**Severity:** Low
**File:** `crates/aptu-coder-core/src/cache.rs` (`lock_shard_shared` call sites)
**Verdict:** CONFIRMED

`lock_shard_shared` returns `Option<ShardLockGuard>` and silently returns `None` on lock acquisition failure, after which callers proceed without the shard lock. This is intentional best-effort advisory locking, but call sites carry no comment clarifying the no-lock case, making the behavior non-obvious to future contributors.

**Resolution:** Fixed in [#1282](https://github.com/clouatre-labs/aptu-coder/pull/1282). Call sites annotated with best-effort comments; `tracing::debug!` added on the no-lock path.

---

### A01 -- AGENTS.md `cfg_attr` claim for `unwrap_used` is inaccurate

**Severity:** Info
**File:** `AGENTS.md`, `Cargo.toml`
**Verdict:** CONFIRMED

AGENTS.md states *"unwrap\_used (test code exempted via cfg\_attr)"*. No `cfg_attr(test, allow(clippy::unwrap_used))` exists in `Cargo.toml` or any per-file module. Test code appears not to use `.unwrap()` in practice. The documented exemption mechanism does not exist.

**Resolution:** Fixed in [#1282](https://github.com/clouatre-labs/aptu-coder/pull/1282). AGENTS.md updated to reflect the `cfg_attr` exemption mechanism in each crate's `lib.rs` rather than `Cargo.toml`.

- `#[tool_router]` on `impl CodeAnalyzer` and `#[tool_handler]` on `impl ServerHandler for CodeAnalyzer` are on separate `impl` blocks as required.
- Every `CallToolResult::success(...)` response carries `.with_meta(Some(no_cache_meta()))`.
- `Content::text()` used exclusively; `RawContent` is absent.
- Transport wiring follows the canonical `stdio() / serve_server() / waiting()` pattern.
- Tool methods correctly take `_context: RequestContext<RoleServer>` as the second parameter.
- No deprecated rmcp APIs detected.

### Core Engine

**Status: Sound.** All findings resolved (F11, F12, F13 fixed in [#1282](https://github.com/clouatre-labs/aptu-coder/pull/1282)).

- BFS traversal in `find_chains_bfs()` correctly caps depth at `follow_depth` and maintains a `visited` `HashSet` to prevent cycles.
- L1 LRU and L2 disk cache key correctness verified (includes mtime, mode, git\_ref). File locking (fs2, PR #1255) applied consistently.
- Formatter split (PR #1233: emit, render, summary, pagination modules) has no dead code or inconsistency.
- Parameter validation for `summary+cursor` mutual exclusion, `impl_only` non-Rust check, and `import_lookup` empty-symbol check all enforced at handler layer with context-aware error messages.
- No `todo!()` or `unimplemented!()` macros in production paths.

### exec\_command

**Status: Sound.** F10 (slot collision) fixed in [#1283](https://github.com/clouatre-labs/aptu-coder/pull/1283).

- All 4 heredoc rejection cases in `shell_write.rs` verified.
- Stdin 1 MB cap enforced; stdin handle closed correctly via RAII.
- Timeout handling correctly sets `exit_code` to `null`.
- `drain_timeout_secs` negative-value rejection enforced before spawn.
- No shell injection vectors; command passed as single `-c` argument.
- PR #1268 (output\_truncated sync to structuredContent) is complete and correct.
- Integration tests cover all major scenarios including interleaved overflow, slot isolation, filter application, and drain behavior.

## Non-Findings (considered and dismissed)

**F01 (was Critical):** `exec_command.rs:447` -- `let output` missing `mut`. Dismissed. Line 479 extracts `output.output_truncated` into a new `let mut output_truncated` binding; line 493 assigns back to the struct field. `cargo build` passes. The validator confirmed this before dismissal.

**F08 (was Low):** `release.yml` `update-homebrew` job lacks explicit `permissions:` block. Dismissed. The top-level `permissions: contents: read` at line 19 is inherited by all jobs. No security gap.

**F09 (was Low):** `exec_command.rs:780` BrokenPipe branch does not explicitly drop `stdin_handle`. Dismissed. Rust RAII drops `stdin_handle` at scope exit. Implicit drop is correct.

**A03 (was High):** `server.rs:180` -- `exec_command` not disabled in `analyze` profile. Initially confirmed by the validation delegate. Subsequent implementation attempts (#1272, #1275) revealed the finding was incorrect: the implementation was not a simple one-line fix, and attempts to patch it exposed that the `APTU_CODER_PROFILE` feature itself was under-specified. The feature was removed entirely in PR [#1278](https://github.com/clouatre-labs/aptu-coder/pull/1278) (`refactor(server): remove APTU_CODER_PROFILE feature`, -481 lines, 153 tests passing). A03 is therefore a false positive on the original finding: the correct resolution was feature removal, not patching a contract violation.
